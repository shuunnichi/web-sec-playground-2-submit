# Web Security Playground 2 - 認証・認可機能の実装

## 概要
本プロジェクトは、Next.jsを用いてセキュアな認証・認可機能をゼロから実装したウェブアプリケーションです。
教材（web-sec-playground-2）をベースに、不要な機能やコードを完全に排除し、「セッションベース認証」に特化した安全な設計を行いました。

## 実装した追加機能（2点）
本課題の要件として、以下の2つの独自機能をフロントエンドおよびバックエンドに実装しました。

### 1. パスワードの表示・非表示切り替え機能
- **実装箇所:** ログイン画面 (`/login`) および サインアップ画面 (`/signup`)
- **詳細:** パスワード入力欄の右端に「👁️（目）」のアイコンを配置し、クリックすることで `<input>` の `type` 属性を `password` と `text` で切り替えられるようにしました。これにより、ユーザの入力ミスの防止とUXの向上を図っています。

### 2. サインアップ時の確認用パスワード要求機能
- **実装箇所:** サインアップ画面 (`/signup`) および Zodバリデーション (`SignupRequest.ts`)
- **詳細:** アカウント登録時に「パスワード（確認用）」の入力を必須としました。Zodの `.refine()` メソッドを用いて、フロントエンドとバックエンドの両方で「2つのパスワードが完全に一致しているか」を厳密にチェックし、不一致の場合はわかりやすいエラーメッセージを表示するよう実装しています。

## セキュリティ対策と工夫した点
「ガチガチにセキュアな設計」を目指し、以下の対策を講じています。

- **パスワードのハッシュ化:** 平文でのパスワード保存を廃止し、`bcryptjs` を用いてハッシュ化（コストファクタ: 10）した上でデータベースに保存しています（`seed.ts`, `signup.ts`）。また、ログイン時の照合にも `bcrypt.compare` を使用しています。
- **セッションベース認証への完全移行とコードのクリーンアップ:** XSSによるトークン流出リスクを考慮し、JWT（トークンベース認証）を無効化しました。さらに、誤動作や予期せぬ脆弱性を防ぐため、使用しないJWT関連のヘルパー関数や、課題に無関係な「ニュース機能」「ショップ機能」関連のAPI・コンポーネント・DBスキーマをすべて削除し、セキュアでクリーンなコードベースを保っています。
